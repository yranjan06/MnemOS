most ai agents are amnesiac. they start fresh every run, repeat the same mistakes, and have no concept of what worked before.

MnemOS gives your agents persistent memory, automatic error recovery, and context-aware decision making — built into every node.

memory
remember node
saves observations, prices, and outcomes to HydraDB. persists across every run and container restart — your agent never forgets what it has seen.
memory
recall node
fetches relevant past memory via graph-enhanced search. surfaces useful context, not just similar text, giving your agent real situational awareness.
recovery
recover node
wraps any action in automatic error handling. checks memory for past failures, asks the llm for a fix, and retries — no manual intervention needed.
adaptation
plan node
reads memory context and picks the best strategy from your options. its decision changes every run based on what actually worked before.
how it works
01.
install in two commands
clone the repo, copy your env file, and bring up the full stack with docker compose. the memory core, agent runtime, and real browser are all included.
$ git clone github.com/yranjan06/MnemOS
$ docker compose up --build
>> ready at localhost:3000
02.
build visually
open the workflow builder and drag nodes onto the canvas. connect them, configure each action, and add memory nodes wherever your agent needs context.
navigate → do → check
remember → recall → plan
>> workflow ready
03.
run and adapt
trigger your agent manually or on a cron schedule. watch it execute in a real browser, recover from errors automatically, and improve with every run.
$ run workflow price_tracker
>> recalled 3 past prices
>> alert threshold met
